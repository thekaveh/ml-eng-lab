ml-eng-lab — Mixture-of-Experts (MoE) Routing

Top-k=2 routing across 4 expert FFNs with Switch-Transformer auxiliary load-balancing loss. Constructed via nnx.MoELinear(in, out, num_experts=4, top_k=2).

Routing path

  • Input x → gating MLP → softmax → top-k mask → weighted combine → y.
  • Only 2 of 4 experts active per token (conditional computation).
  • Each token can take a different path through the expert pool.

Sparsity & combine

  • num_experts=4, top_k=2 ⇒ 50% of experts active per forward pass.
  • Inactive experts' outputs masked to 0 — no contribution to Σ wᵢ·Eᵢ(x).
  • Combine y = Σᵢ wᵢ · Eᵢ(x) sums only the selected top-k outputs.

Load balancing

  • L_aux = N · Σᵢ fᵢ·Pᵢ (Switch Transformer) added to training loss.
  • fᵢ = token fraction, Pᵢ = gate-prob mass for expert i.
  • Prevents expert collapse where one expert dominates routing.
