- `self_supervised-fmnist-jepa-pytorch/` ‚Äî new Tier-A single-notebook task. First in-repo demo of the nnx megamerge I-JEPA self-supervised stack: `ViTNN(image_size=28, patch_size=4, in_channels=1, d_model=64, n_layers=2, n_heads=4)` context encoder (~103k params); `build_target_encoder(model.net)` for the frozen EMA target; `JEPAPredictor(embed_dim, n_patches, predictor_dim=32, n_layers=2, n_heads=2)` (~30k params) attached to `model.net` so the optimizer picks up its params jointly; `random_block_mask(n_patches, grid_size=7)` for the context/target patch split; `jepa_train_step_factory(target_encoder, predictor, mask_fn, ema_momentum=0.996)` to wire everything together. Phase 2: `LinearProbe(encoder, embed_dim, num_classes)` freezes the pretrained encoder and trains only a `Linear(64 ‚Üí 10)` head on top of mean-pooled patch embeddings ‚Äî the standard SSL evaluation. Substituted Fashion-MNIST for the paper's ImageNet / example's CIFAR-10 (already in the collection, CPU-feasible at Tier-A budget). Recorded run (~90 s CPU, 2 JEPA epochs + 5 probe epochs): JEPA pretrain loss 1.0416 ‚Üí 0.3162; linear-probe val accuracy 70.62% ‚Üí 74.43% (well above the 10% random baseline ‚Äî SSL proof-of-life). - `link_prediction-karate-graphsage-pyg/` ‚Äî new Tier-A single-notebook task. Link-prediction sibling to `node_classification-reddit-gnn-pyg/`: same GraphSAGE architecture but used as an *unsupervised edge encoder* on Zachary's Karate Club (34 nodes, 78 undirected edges, identity features). 2-layer `SAGEConv` (34‚Üí32‚Üí16) encoder + dot-product edge scorer `œÉ(z_u ¬∑ z_v)`, trained with BCE on positive observed edges + fresh-resampled negative non-edges per epoch via `torch_geometric.utils.negative_sampling`. Edge splits via `RandomLinkSplit(num_val=0.1, num_test=0.2)`. Recorded run (~5 s CPU, 200 epochs): train BCE 0.7101 ‚Üí 0.3060, val AUC 0.673 ‚Üí 0.735, test AUC noisy (~0.43 on the recorded split; the ¬ß6.3 prose owns this ‚Äî the 30-edge test set has variance of ~10 pp). - `diffusion-mnist-ddpm-pytorch/` ‚Äî new Tier-A single-notebook task. First in-repo demo of the nnx megamerge diffusion stack: `NoiseSchedulers.LINEAR(T=100)` for the schedule, `DiffusionMLP(input_dim=784, hidden_dims=[256, 256], time_embed_dim=32)` for the denoiser (~477k params), `diffusion_train_step_factory(schedule)` as the `train_step_fn`, `sample(model, schedule, shape=(16, 784))` for the reverse-diffusion sampling loop. - `tabular_regression-diabetes-mlp-pytorch/` ‚Äî new Tier-A single-notebook task. **First regression task in ml-lab.** sklearn `load_diabetes` (442 samples √ó 10 features, continuous target). - `dim_reduction-iris-autoencoder-pytorch/` ‚Äî new Tier-A single-notebook task. PCA baseline vs two autoencoder variants (shallow `[2]`-bottleneck and deeper symmetric `[3, 2, 3]`) on iris, projected to a 2-D latent. First in-repo demo of `nnx.NNModel.train(..., train_step_fn=autoencoder_step)` outside the transformer LM task ‚Äî `autoencoder_step` implements `MSE(net(X), X)` and ignores the dummy `y`. Both AEs use the `FeedFwdNN(input_dim == output_dim, hidden_dims=...)` trick ‚Äî no custom `nn.Module` subclass, the bottleneck is the middle Linear. Recorded run (300 epochs, ~18 s CPU): PCA explained-variance ratio [0.83, 0.13], reconstruction loss shallow `[2]` 0.5722 ‚Üí 0.0142, deeper `[3, 2, 3]` 0.4212 ‚Üí 0.0492. Held-out test linear-probe accuracy (LogisticRegression on the 2-D latent): PCA 82.61%, AE shallow `[2]` 100.00%, AE deeper `[3, 2, 3]` competitive. ¬ß6.3 owns the data-dependent trade-off: at iris scale the extra capacity in `[3, 2, 3]` can overfit the reconstruction and lose species separation. - `peft-mnist-to-fmnist-dora-vs-lora-pytorch/` ‚Äî new Tier-A single-notebook task. First in-repo demo of `nnx.apply_lora_to` and `nnx.apply_dora_to` for cross-task adapter fine-tuning. Pretrains a small FFN (`hidden_dims=[128, 64]`, `109,386` total params) on MNIST for 2 epochs, snapshots the state_dict, then spins up three Fashion-MNIST adapt paths from the *same* pretrained base: (a) full fine-tune control (all params trainable, 109,386), (b) LoRA at `r=8`, `alpha=16` (`9,424` trainable, 8.6% of base; wraps 3 Linear layers), (c) DoRA at same rank/alpha (`9,626` trainable, +202 magnitude params vs LoRA). Also exercises `save_lora_weights` / `load_lora_weights` round-trip into a fresh adapter (39.2 KB checkpoint file vs ~440 KB of base weights). ¬ß6.2 calls out the budget-dependent trade-off: at the short 2-epoch-pretrain + 1-epoch-adapt schedule the full-fine-tune control still leads on val accuracy; PEFT wins land at longer horizons on well-converged bases. ~25 s on CPU.
- `text_generation-tinyshakespeare-transformer-pytorch/` ‚Äî new Tier-A single-notebook task. First in-repo end-to-end demo of the nnx transformer fork: trains a 256-vocab BPE tokenizer (`nnx.train_bpe` / `NNTokenizerParams`) on an embedded ~7 KB Shakespeare corpus (tiled 8√ó for loader-size + BPE frequency), tokenizes the corpus into a fixed-length window dataset (`SEQ_LEN=32`, `y = x.roll(-1)` next-token targets), builds a ~114 k-param `TransformerNN` (`d_model=64`, `n_heads=4`, `n_layers=2`, `max_seq_len=32`, default RoPE + tied embeddings), trains via `model.train(train_step_fn=lm_train_step)` with a custom step that flattens `(B, T, V) ‚Üí (B*T, V)` for `cross_entropy`, then samples generations via `GenerativeNNModel.generate(prompt, max_new_tokens=32, temperature=0.8, top_k=20, seed=42)` for three Shakespeare-style prompts. At this scale + budget the generations are not coherent ‚Äî that's the ¬ß6.3 "correctness smoke test, not quality" framing, with scaling levers (bigger corpus, bigger model, real TinyStories via HF) called out. ~8 s on CPU.
- `knowledge_distillation-mnist-ffnn-pytorch/` ‚Äî new Tier-A single-notebook task. Born-again training demo (Furlanello et al., 2018) via `nnx.born_again_train`: trains a single-gen FP32 baseline as the reference, then separately runs a 3-generation born-again chain from the same fresh init (gen 0 trains on hard labels; gen k>0 distills from a frozen deepcopy of gen k-1 using `nnx.kd_train_step_factory(teacher=...)`). The recorded run shows the per-generation val-loss trajectory (2.14 ‚Üí 1.91 ‚Üí 1.65) and a +21% accuracy delta vs single-gen at the same parameter count, attributable to the soft-label regularization effect. ~25 s on CPU.
- `pruning-mnist-ffnn-pytorch/` ‚Äî new Tier-A single-notebook task. First in-repo demo of `nnx.prune.magnitude_prune`: trains an FP32 FFN baseline (`hidden_dims=[256, 128]`), then deep-copies + prunes at each target sparsity `s ‚àà {0.1, 0.3, 0.5, 0.7, 0.9}`, measuring validation accuracy + actual zero fraction. - `quantization-mnist-ffnn-pytorch/` ‚Äî new Tier-A single-notebook task.
